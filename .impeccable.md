## Design Context

### Users
Knowledge workers, researchers, and developers who want a clean, local-first markdown editor with rich rendering (math, diagrams, code) and full control over appearance. They value speed, keyboard-driven workflows, and owning their files — no cloud lock-in, no proprietary formats.

### Brand Personality
**Crafted, thoughtful, quiet.** RustyNotes feels hand-made and intentional — like a precision tool built by someone who cares deeply about the craft. The rust-toned identity gives it warmth without being loud. It should feel engineered with care, not manufactured.

### Emotional Goals
**Technical precision** — sharp, efficient, built for people who notice the details. Users should feel in command of a finely tuned instrument, not wrestling with software.

### Aesthetic Direction
- **Visual tone:** Quiet minimalism with technical depth. Clean surfaces, tight typography, purposeful spacing. Every pixel earns its place.
- **References:** VS Code / Zed — keyboard-driven, developer-oriented, fast, extensible. The density and capability of a code editor, but with the warmth of a writing tool.
- **Anti-references:** No Electron bloat (must feel native and fast), no Google Docs (no generic web-app feel), no plain-text-editor bareness (rich rendering and polish matter).
- **Theme:** Dark mode as primary experience (Catppuccin Mocha base), light mode as equal citizen. System auto-detection with user override.

### Design Principles

1. **Content is king** — The UI exists to serve the writing. Chrome should recede; markdown content should dominate. 65ch line width, generous vertical rhythm, typography-first layout.

2. **Fast and honest** — No loading spinners disguising sluggishness. Transitions are functional (0.2s max), not decorative. If something is instant, show it instantly. Tauri + Solid.js means there's no excuse for jank.

3. **Precision over decoration** — Spacing is mathematical (4/8/12/20/32/48 scale). Colors are systematic (semantic tokens, not ad-hoc hex values). Every visual choice should be traceable to a design token.

4. **Keyboard-first, mouse-welcome** — Power users live on the keyboard (Cmd+1/2/3, Cmd+E, Cmd+K). But every keyboard action has a discoverable mouse equivalent. No hidden-only shortcuts.

5. **Earn complexity** — Start simple, reveal depth. Settings panel slides in on demand. Navigation modes switch cleanly. Advanced rendering (math, diagrams) loads lazily. The app feels simple until you need it not to be.

### Technical Design Constraints
- **Font stack:** System fonts (-apple-system, SF Mono) — no web font loading
- **Color system:** CSS custom properties applied via JS from JSON theme files
- **Spacing:** 6-step scale (xs:4, sm:8, md:12, lg:20, xl:32, 2xl:48)
- **Border radius:** 4px (subtle), 6px (buttons/inputs), 8px (panels/cards)
- **Icons:** Unicode symbols only — no icon font or SVG sprite sheet
- **Motion:** Max 0.2s transitions, respect prefers-reduced-motion
